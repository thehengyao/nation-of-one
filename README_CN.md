# 🏛️ 一人之国 Nation of One

![Nation of One](assets/cover.jpg)

> [English README](README.md)

---

大多数使用效率工具的人，其实都在做执行的工作。任务、计时器、打卡连续天数。有用——直到某天开始感觉哪里不对。

真正的问题是：执行工具默认你已经知道自己在为什么执行。也默认做执行这件事的人是一个稳定的、可以被优化的单位。这两个前提都不成立。

这个问题在 vibe coding 圈子里也很明显。有人开始做产品，第一个项目是 todo app，第二个是习惯追踪，第三个是番茄钟。不是因为这些是最值得做的东西，而是因为他们脑子里运行的那个心智模型就是这个形状——生活是一个需要清空的任务队列。

但人不是队列。人是一个不断展开的过程。目标从来不是完成更多，而是更成为你自己。

**Nation of One 从这里出发。**

你不是一条需要被优化的流水线。你是一个需要被治理的国家——有外交部管你的关系，有国防部守你的情绪边界，有情报局追踪你真正在关注什么。八个部委，一种人生，同时运作。

---

## 实际上是什么样的

一个设计师背景的独立创作者，同时推进三个项目。其中一个有起色，两个停滞。她已经连续记录六周了。

她的 NDRC（目标）一直是 B，但她感觉自己很忙。MOF（时间和精力）的数据显示，她 60% 的时间花在了那两个停滞的项目上。她的 MOD 顾问——一个以不让你回避不舒服的事著称的人——指出了一个她没有命名过的模式：她倾向于往那些自己掌控输出的项目跑，而不是需要她被看见的那个。

她不需要再一个任务管理工具。她需要看清楚那个模式。

这就是 Nation of One 做的事。

## 八个部委

| 代码 | 部委 | 管什么 |
|------|------|--------|
| NDRC | 国家发展和改革委员会 | 目标、OKR、重大决策 |
| MOF | 财政部 | 钱、时间、精力分配 |
| MOE | 教育部 | 学习、技能、有深度的输入 |
| NHC | 国家卫生健康委员会 | 睡眠、运动、饮食、身体信号 |
| MFA | 外交部 | 人脉、关系、社交能量 |
| MCT | 文化和旅游部 | 休息和娱乐——不需要理由 |
| MOD | 国防部 | 情绪韧性、风险管理、边界守护 |
| NIA | 国家情报局 | 行业前沿 + 兴趣圈追踪 |

## 核心功能

**普通建国**
回答几个问题，确定你的国家类型、发展阶段和各部委基线评级。当天就能开始运作。

**深度建国**
三种入口：
- *你有多年的原始数据* — 日记、笔记、聊天记录。喂进来。系统会用八个部委的视角构建一个持续积累的知识库（参考 [Andrej Karpathy 的 LLM wiki 框架](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f)），然后生成一份六维自我画像：优势区、盲点区、核心张力、能量地图、机会窗口、风险轨道。
- *你有一些材料* — 有多少用多少。画像会是局部的，但会如实告诉你。
- *你什么都没有* — 从普通建国开始，边用边积累。

**每日提报**
快速过一遍八个部委，只记异常，正常的不用写。

**主动巡视**
每次对话开始前，系统自动扫描：几天没提报、哪个部委持续低分、顾问有没有新情报。不用你开口。

**内阁会议**
周报和月报，S/A/B/C/D 评级加趋势箭头。

**顾问团**
把真实的人蒸馏成顾问人格，任命到对应部委，需要的时候召见。
[nuwa-skill](https://github.com/alchaincyf/nuwa-skill)（作者 [@alchaincyf](https://github.com/alchaincyf) · [X](https://x.com/alchainhust)）负责把人蒸馏出来，Nation of One 负责让他们上岗。

**情报简报**
联网搜索你的专业领域和兴趣方向，情报部真正在工作。

## 怎么用

| 你说 | 发生什么 |
|------|---------|
| `"建国"` | 普通建国流程 |
| `"深度建国"` / `"导入自我"` | rawdata → 自我画像 → 顾问推荐 |
| `"汇报"` | 每日部委提报 |
| `"开内阁会议"` | 周度评级和趋势分析 |
| `"月报"` | 月度宏观报告 |
| `"情报简报"` | 联网搜索你的关注领域 |
| `"定目标"` | 战略规划和 OKR |
| `"财政部怎么样"` | 单部委深度分析 |
| `"蒸馏 [人名]"` | 把一个人提炼成顾问人格 |
| `"任命 [人名] 为 [部委]"` | 将顾问任命到对应部委 |
| `"召见 [人名]"` | 召见顾问，以其视角回应当前问题 |
| `"顾问团状态"` | 查看所有顾问和任命情况 |

中文、英文、中英混合都可以。

## 数据存储

所有数据存在 `~/Documents/nation/`，纯 Markdown，本地运行，不上云，全部属于你。

```
~/Documents/nation/
├── founding.md        # 建国宣言
├── config.md          # 部委配置 + 顾问任命
├── goals.md           # OKR（你准备好了再定）
├── logs/              # 每日提报
├── weekly/            # 内阁会议记录
├── monthly/           # 月度宏观报告
├── wiki/              # 从你的数据建立的知识库
└── advisors/          # 蒸馏好的顾问人格档案
```

## 设计原则

1. **先诊断，后规划** — 搞清楚你的国家现状，再谈目标
2. **休息就是休息** — 文旅部不等于教育部，娱乐不需要合理化
3. **数据极简** — 只记异常，正常不写
4. **主权在你** — 所有建议都是参考，元首说了算
5. **没有盲区** — 系统如实反映数据里看到的，包括你可能不想看到的
